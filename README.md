# 一、简介
LFBetterAudio，一款适用于游戏《学生时代》，基于 BepInEx、Harmony 和 Unity 的游戏音频演出插件。

插件面向Modder和玩家，采取类似原版的Data-Driven的形式，自定义 `1163` EFFECT，为剧情提供独立于原版音频系统的播放与演出能力，并支持如下内容：
- 单 Talk /跨 Talk 的背景/唱歌音乐演出
- 音效演出
- 可扩展双语的浮动歌词
- 指定句段播放
- 含淡入淡出的暂停、恢复功能
- 32 色歌词调色板
- 歌词位置拖动、复位与锁定

BetterAudio 同时兼容正常游戏剧情和剧情编辑器 Preview。

*<font color="#4bacc6">此模组实际代码编写工作由AI完成，本人仅负责构思及debug</font>*

# 二、安装方法
## 2.1 前置需求
- 《学生时代》游戏本体
- 与游戏当前 Mod 环境匹配的 BepInEx 5.x
- （推荐）启用 BepInEx 控制台或日志文件，便于查看启动自检结果
## 2.2 安装插件
1. 安装并确认 BepInEx 可以正常加载。
2. 获取编译后的 `LFBetterAudio.dll`。
3. 将 DLL 放入：
```text
<游戏目录>/BepInEx/plugins/
```
4. 启动游戏。
5. （推荐）检查 BepInEx 控制台或日志。

正常启动时应看到类似内容：
```text
[启动自检成功] BetterAudio 1.0.0 全流程自检通过；音频条目=...，资源包=...。
```

若出现：
```text
[启动自检失败] BetterAudio 1.0.0 全流程自检未通过：...
```
请根据后面的具体错误检查引用、资源包、关键补丁或运行环境。

## 2.3 Mod 作者模板
插件启动后会在以下位置维护一份作者模板：
```text
<游戏目录>/BepInEx/plugins/BetterAudio/
```
该目录仅供复制和参考，**不会被当作正式音乐资源包注册**。

正式发布时，应把整个 `BetterAudio` 文件夹放入自己的 Mod 根目录：
```text
<你的 Mod 根目录>/
└─ BetterAudio/
   ├─ Audio/
   ├─ Timeline/
   └─ BetterAudio.json
```

上传到 Steam 创意工坊后，BetterMusic 会直接扫描并读取对应 Workshop Item，无需要求玩家手动复制音乐和歌词到 BepInEx。

## 2.4 接入StudentAgeModManager
本插件已接入StudentAgeModManager。
项目地址：https://github.com/white12666/StudentAgeModManager
如正确安装了modmanager，那么只需要通过steam创意工坊订阅，随后像普通的mod一样在游戏中启用即可。


# 三、资源包配置
## 3.1 标准目录
```text
BetterAudio/
├─ BetterAudio.json
├─ Audio/
│  ├─ example.mp3
│  └─ ...
└─ Timeline/
   ├─ example.lrc
   └─ ...
```
文件夹和配置文件名称不区分大小写，但建议统一使用上述写法。

## 3.2 `Betteraudio.json`
示例：
```text
{
  "audios": [
    {
      "id": 90001,
      "name": "自定义音乐示例",
      "audioPath": "Audio/example.mp3",
      "timelinePath": "Timeline/example.lrc",
      "volume": 0.85,
      "type": 1
    },
    {
      "id": 90002,
      "name": "自定义音效示例",
      "audioPath": "Audio/example_sfx.mp3",
      "timelinePath": "",
      "volume": 1.0,
      "type": 2
    },
    {
      "id": 12345,
      "name": "原版音频外挂 Timeline 示例",
      "audioPath": "",
      "timelinePath": "Timeline/original_12345.lrc",
      "volume": 1.0,
      "type": 1
    }
  ]
}
```
字段说明：

| 字段             | 是否必填 | 说明                          |
| -------------- | ---: | --------------------------- |
| `id`           |    是 | 音频 ID，必须大于 0，并避免与其他 Mod 冲突  |
| `name`         |   建议 | 日志中显示的音频名称                  |
| `audioPath`    |  视情况 | 自定义音频相对路径；留空时尝试使用同 ID 的原版音频 |
| `timelinePath` |    否 | `.lrc` Timeline 相对路径；可留空    |
| `volume`       |    否 | 播放音量                        |
| `type`         |    否 | `1=音乐`，`2=音效`；缺省或非法值按 `1`   |
相对路径以当前 Mod 自己的 `BetterAudio` 文件夹为基准。
## 4.3 音频解析优先级
1. 内置保留音频 `1163001`；
2. 创意工坊或 Mod 目录中的 `BetterAudio.json`；
3. 编辑器 Preview 当前音频配置；
4. 正常游戏原版 `AudioCfg`。

原版音频会读取原版 `AudioCfg.type`：
- `type=1` 识别为音乐；
- `type=2` 识别为音效；
- 其他值按音乐处理。

# 四、1163 EFFECT 总览

| 指令                                    | 功能                  |
| ------------------------------------- | ------------------- |
| `1163,0`                              | 停止当前音乐和全部插件音效，并刷新通道 |
| `1163,1,x,y,z,m[,u[,v]]`              | 单 Talk 背景型音乐        |
| `1163,2,x,y,z,id1,id2...,-1[,u[,v]]`  | 单 Talk 唱歌型音乐        |
| `1163,3,x,y[,u[,v]]`                  | 单 Talk 音效           |
| `1163,10,x,y,z,m[,u[,v]]`             | 跨 Talk 背景型音乐        |
| `1163,20,x,y,z,id1,id2...,-1[,u[,v]]` | 跨 Talk 唱歌型音乐        |
| `1163,30,x,y[,u[,v]]`                 | 跨 Talk 音效           |
| `1163,99,1`                           | 暂停当前音乐              |
| `1163,99,2`                           | 恢复当前音乐              |
| `1163,99,3`                           | 暂停最近建立且仍有效的插件音效     |
| `1163,99,4`                           | 恢复最近暂停的插件音效         |
| `1163,99,10`                          | 音乐淡出后暂停             |
| `1163,99,20`                          | 音乐淡入恢复              |
## 4.1 类型限制
以下指令仅接受 `type=1`：
```text
1163,1
1163,2
1163,10
1163,20
```
如果传入 `type=2`，效果不执行并输出错误日志。

以下指令仅接受 `type=2`：
```text
1163,3
1163,30
```
如果传入 `type=1`，效果不执行并输出错误日志。
## 4.2 批次确定性
同一 Talk 中的 `1163` 分两批执行：
```text
第一批：播放型指令 1 / 2 / 3 / 10 / 20 / 30
第二批：控制型指令 0 / 99,x
```
因此“播放后暂停”具有稳定结果。若音频仍在加载，将使用 `PendingPauseAfterLoad`，加载完成后直接暂停，不先播放声音或闪现歌词。
## 4.3 多EFFECT的处理
- 一次只能存在一个插件音乐会话；
- 可以同时存在多个插件音效会话；
- 音乐与音效可以共存；
- 暂不保证同一 Talk 中两个音乐播放指令的组合行为；
- 当前音效暂停/恢复指向最近建立且仍有效的插件音效。

# 五、停止、暂停与淡入淡出
## 5.1 停止并刷新
```text
1163,0
```
作用：
- 停止当前音乐；
- 停止全部插件音效；
- 清理歌词和长按跳过提示；
- 释放所有相关会话；
- 归还原版 BGM 通道；
- 即使当前没有插件音频也可使用，等同于刷新插件音频状态。
## 5.2 音乐暂停与恢复
```text
1163,99,1
1163,99,2
```
暂停后记录播放位置、暂停插件音乐、清理当前歌词并归还原版 BGM；恢复后从原位置继续，重新取得音乐优先级并同步歌词。
## 5.3 音效暂停与恢复
```text
1163,99,3
1163,99,4
```
仅针对最近建立且仍有效的插件音效，不影响音乐和其他较早音效。
## 5.4 音乐淡出与淡入
```text
1163,99,10
1163,99,20
```
- `1163,99,10`：使用平滑曲线将音乐音量降至 0 后暂停；
- `1163,99,20`：从暂停点恢复，并从 0 平滑恢复到目标音量；
- 淡变使用非缩放时间，避免快进和 `Time.timeScale` 干扰；
- 新播放、停止或其他控制指令会取消旧淡变任务，避免多个淡变同时修改音量。
# 六、背景型音乐
## 6.1 单 Talk 背景型音乐
```text
1163,1,x,y,z,m[,u[,v]]
```

| 参数 | 说明 |
|---|---|
| `x` | `type=1` 的音乐 ID |
| `y` | 播放类型，只允许 `1`、`2`、`3` |
| `z` | 歌词字号：`-1`、`1`、`2`、`3`、`4` |
| `m` | 歌词颜色：`0～31` |
| `u` | 可选，起始 Timeline 记录序号 |
| `v` | 可选，结束 Timeline 记录序号 |

| `y` | 行为                                   |
| --: | ------------------------------------ |
| `1` | 单次播放；400ms 后允许手动推进；播放完成前自动模式不推进；快进失效 |
| `2` | 循环播放；手动推进后停止；自动模式不推进；快进失效            |
| `3` | 必须播放完成或长按跳过；手动、自动和快进均按强制演出规则处理       |
|     |                                      |

## 6.2 跨 Talk 背景型音乐
```text
1163,10,x,y,z,m[,u[,v]]
```
`y`表示播放次数：
- `y=0`：无限循环；
- `y>=1`：总播放次数；
- 非法负数拒绝执行。

其他规则：
- 进入下一 Talk 后继续保持；
- 不强制依赖 `1163,0` 才能结束；
- 有限次数播放完后自动停止并归还原版 BGM；
- 仍可随时使用 `1163,0`、暂停指令或新音乐指令改变状态；
- 不阻止自动模式和快进。
# 七、唱歌型音乐
## 7.1 单 Talk 唱歌型音乐
```text
1163,2,x,y,z,id1,id2,id3...,-1[,u[,v]]
```
规则：
- `x` 必须为 `type=1`；
- `y` 强制为 `3`；
- `z` 只允许 `1～4`，非法值按 `1`；
- `id1、id2...` 为演唱角色实际 roleid；
- `-1` 表示角色列表结束；
- 音乐必须播放完成或由玩家长按跳过；
- 自动模式和快进不生效；
- 只对当前 Talk 生效。
## 7.2 跨 Talk 唱歌型音乐
```text
1163,20,x,y,z,id1,id2,id3...,-1[,u[,v]]
```
规则：
- `y` 强制为 `1`；
- 单次播放；
- 进入下一 Talk 后继续保持；
- 音乐完成前阻止自动推进；
- 玩家手动推进有效；
- 快进有效。
## 7.3 演唱角色槽
Timeline 中的 `id1`、`id2` 表示 EFFECT 角色列表中的位置，并不是直接 roleid。
```text
1163,2,90001,3,1,103,0,-1
```
表示：
- `id1` → roleid `103`；
- `id2` → roleid `0`；
- roleid `0` 代表玩家。
Timeline 使用了超出角色列表的 `idN`，或没有 `idN` 前缀时，该行按“合唱”处理。

# 八、音效
## 8.1 单 Talk 音效
```text
1163,3,x,y[,u[,v]]
```
- `x` 必须为 `type=2`；
- `y=1`：单次播放；
- `y=2`：循环播放；
- 不显示歌词；
- 不使用 `z` 和 `m`；
- 不抢占原版 BGM；
- 可与音乐及其他插件音效同时播放。

`y=1`：
- 不干扰玩家手动推进；
- 自动推进至少等待音效播放完成；
- 不干扰快进。
`y=2`：
- 不干扰玩家手动推进；
- 自动推进失效；
- 不干扰快进；
- 单 Talk 结束时停止。
## 8.2 跨 Talk 音效
```text
1163,30,x,y[,u[,v]]
```
参数含义与 `1163,3` 相同，但音效可跨 Talk 保持。

# 九、推进规则叠加
音乐和音效的推进限制独立计算，并最终合并。
例如同一 Talk 使用：
```text
1163,1,90001,1,1,0
1163,3,90002,1
```
最终效果：
- 音乐和音效各播放一次；
- 玩家手动推进按音乐规则处理；
- 自动推进必须等待音乐、音效及原版其他条件全部解除；
- 音效本身不阻止快进，但音乐会阻止，因此最终快进仍失效；
- 音效不会抢占原版 BGM，音乐会取得优先控制。

# 十、Timeline 与 `u/v`
## 10.1 Timeline 的定位
`Timeline` 文件夹当前仍存放 `.lrc` 文件。名称虽由 Lyrics 改为 Timeline，但本版本不增加新的 Timeline 事件语法。

`.lrc` 在表现层可作为歌词，在程序层也承担时间轴功能：
- 时间标签决定歌词切换；
- `u/v` 使用有效时间记录定位区间；
- 音效可以使用 `.lrc` 时间标签限制播放区间，但不显示文本。

没有配置 Timeline 时：
- 未填写 `u/v` 的音频仍可正常播放；
- 使用本应显示歌词的音乐指令时，只播放音频，不显示歌词；
- 填写 `u/v` 时必须存在有效 Timeline，否则指令拒绝执行。

## 10.2 `u/v` 通用规则
- `u=0`：从音频开头开始；
- `u>=1`：从第 `u` 条有效 Timeline 时间记录开始；
- `v` 为包含式终点；
- 实际停止点取第 `v` 条之后下一条更晚的时间记录；
- 循环模式只在选定区间内循环；
- `v` 必须大于等于 `u`；
- `v=0` 无效；
- 越界时新指令拒绝执行；
- 新指令校验失败时，不会提前破坏当前有效会话。

音乐示例：
```text
1163,1,90001,1,1,0,2,5
```

音效示例：
```text
1163,3,90002,1,2,5
```

唱歌型音乐的 `u/v` 位于角色列表终止符 `-1` 之后：
```text
1163,2,90001,3,1,103,0,-1,2,5
```

# 十一、Timeline `.lrc` 格式
## 11.1 标准时间歌词
```text
[00:31.65]第一句歌词
[00:35.47]第二句歌词
```
## 11.2 加入演唱角色槽
```text
id1[00:31.65]第一位角色演唱
id2[00:35.47]第二位角色演唱
```
## 11.3 双语歌词
使用第一个半角竖线 `|` 分隔原文和译文：
```text
id1[00:31.65]原文|译文
```
## 11.4 元数据
无 LRC 时间标签的 JSON/NDJSON 元数据会自动跳过，不计入 `u/v` 序号。
## 11.5 全局时间偏移
```text
[offset:500]
```
单位为毫秒。

# 十三、浮动歌词与交互框
## 13.1 基础显示
- 使用 TextMeshPro 浮动歌词层；
- 支持单语与双语；
- 歌词淡入并轻微向上浮动；
- 使用随字号变化的阴影；
- 根据局部背景亮度自动切换黑色或白色描边。
## 13.2 跨 Talk 防闪烁
背景会话进入下一 Talk 时，如果当前仍是同一条歌词：
- 直接恢复稳定显示；
- 不重新播放淡入和上浮动画；
- 只有真正切换到下一条歌词时才播放动画。
## 13.3 工具栏
点击歌词后显示半透明灰框及三个按钮：
1. **设置**：打开字号与颜色菜单；
2. **复位**：逆时针回转箭头图标，恢复当前 EFFECT 的初始字号和颜色，保留当前位置；
3. **位置锁**：
   - 未锁定时显示开锁图标；
   - 锁定后显示闭锁图标。
图标由代码运行时生成透明纹理并转换为 Sprite，风格和背景保持统一。
## 13.4 位置锁
锁定后：
- 保存当前歌词位置；
- 禁止拖动；
- 灰框保持显示；
- 新播放型 `1163` 继续使用锁定位置；
- 字号和颜色仍按新指令初始化。

解锁后：
- 恢复拖动；
- 恢复自动隐藏；
- 后续新播放指令重新使用默认位置逻辑。
## 13.5 自动隐藏
未锁定时，灰框约在 4.5 秒无操作后隐藏。点击按钮、选择菜单、拖动和结束拖动会重置倒计时。
# 十四、双音频通道与优先控制
## 14.1 音乐
插件音乐使用独立 AudioSource。只有音乐实际开始播放时才暂停原版 BGM。
以下情况会归还原版 BGM：
- `1163,0`；
- `1163,99,1` 或淡出暂停；
- 单 Talk 音乐被推进；
- 长按跳过；
- Talk 关闭或失效；
- 单次音乐自然结束；
- 资源加载或播放失败；
- AudioSource 失效；
- 插件退出。
## 14.2 音效
插件音效使用独立 AudioSource：
- 不暂停原版 BGM；
- 不取得音乐优先权；
- 可与原版音乐、插件音乐和其他插件音效同时播放。

# 十七、常见问题
## Q：控制台出现 `Effect Not Found` 怎么办？
A：正常情况下插件的复核机制会防止该问题。请先查看启动日志是否为 `[启动自检成功]`。若自检失败，应根据错误检查游戏版本、Harmony 补丁目标和插件引用。
## Q：为什么插件音乐停止后原版 BGM 没有恢复？
A：插件处理双通道让权。请确认使用的是新版本，并查看是否出现“原版音乐通道恢复失败”的 `[1163错误]`。也要检查是否有其他 Mod 同时控制原版 BGM。
## Q：为什么填写了 `u/v` 后指令没有执行？
A：请检查：
- ID是否正确，以及LRC是否存在有效时间标签；
- `u` 是否为非负数；
- `v` 是否大于等于 `u`；
- `v` 是否大于 0；
- 句数是否超出有效歌词数量。
## Q：为什么音乐接续有明显的卡顿？
A：这是游戏原版设计就存在的问题，因为游戏原版设计在自动模式推进talk时天然存在间隔，若想尽可能减少卡顿影响，个人建议将文本显示速度和自动播放间隔都调为“快”，至少要是“正常”，不建议设置为“慢”。
## Q：为什么使用了阻碍自动推进的effect，但是自动模式依然会很快速的推进？
A：这是游戏原版设计就存在的问题，只要频繁改动文本显示速度和自动播放间隔，就会导致原版游戏残留大量未清理数据，引起逻辑错误，建议重启游戏。
## Q：为什么唱歌角色显示成“合唱”？
A：可能原因包括：
- LRC 没有 `idN`；
- `idN` 超过 EFFECT 中提供的角色数量；
- roleid 无法解析；
- 对应角色不存在。
这些情况按设计回退为“合唱”。
## Q：为什么背景型音乐没有歌词？
A：检查：
- `z` 是否为 `-1`；
- JSON 是否配置 `timelinePath`；
- LRC 文件是否存在；
- LRC 是否包含有效时间标签；
- `u/v` 是否越界。
## Q：为什么类型不匹配？
- `1163,1/2/10/20` 只适用于 `type=1`；
- `1163,3/30` 只适用于 `type=2`。
## Q：能否只给原版音乐增加歌词？
A：可以。在 JSON 中使用原版音乐 ID，把 `audioPath` 留空，只填写 `timelinePath`。
## Q：没有 Timeline，可以播放吗？
可以。只要音频 ID 和 `audioPath` 或原版 AudioCfg 有效，且没有使用 `u/v`，音频就能播放。音乐指令要求显示歌词时，表现为只播放音频、不显示歌词。
## Q：编辑器 Preview 中能否测试？
A：可以。插件为普通文本 Talk 和空文本 Talk 分别提供了 Preview 桥接。但编辑器版本或底层方法签名变化仍可能影响兼容性。
## Q：插件会修改存档吗？
A：项目没有设计新的存档字段，也不主动改写游戏进度。出于非官方 Mod 的通用风险，仍建议保留备份。
# 十八、维护原则
1. 不要删除 `Plugin.cs` 中的关键 EFFECT 保活和非退出销毁保护。
2. 不要把插件音乐 AudioSource 与原版 BGM AudioSource 合并。
3. 音乐和音效必须保持不同的优先控制规则。
4. 指令格式统一在 `BetterAudioEffectEncoding` 中解析。
5. 音乐状态统一进入 `BetterAudioSession`，音效状态进入 `BetterAudioEffectSession`。
6. 正常游戏和 Preview 必须同时回归测试。
7. 单 Talk 与跨 Talk 生命周期必须分离。
8. 播放型和控制型 EFFECT 必须保持批次确定性。
9. 作者模板不是运行时资源包。
